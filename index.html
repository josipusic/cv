Interviewer:

Could you tell us more about the last feature you have developed?



Josip:

Certainly! I worked on a phonebook uploader feature. Feature was already developed, but was tightly coupled with the rest of the Django application and needed to be refactored.
When user uploads a file it is stored in AWS S3 bucket. Processing lambda function is triggered to validate the data. Data processing is done with pandas library. After data is validated, it is stored in PostgreSQL table which is partitioned by file that is uploaded. User can see processed data and modify it if needed. After user modifies the data, another lambda function is triggered to validate data again. This process is iterative. When user is satisfied, data is saved to permanent table and the processed partitioned table is destroyed.
This way web server is decoupled from the data upload and processing.
